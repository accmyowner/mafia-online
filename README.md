# Мафия Online

Браузерная «Мафия» на чистом JavaScript: ES-модули, Firebase в роли сервера,
ни одной npm-зависимости и ни одного сборщика. Открывается двойным щелчком по
`index.html` (через локальный сервер) и разворачивается на Firebase Hosting одной командой.

---

## Быстрый старт

Модули не работают по протоколу `file://`, поэтому нужен любой локальный сервер:

```bash
cd mafia-online
python3 -m http.server 8080
# или: npx serve .   /   php -S localhost:8080
```

Откройте `http://localhost:8080`.

Пока `firebase/config.js` не заполнен, игра запускается **в локальном режиме**:
комнаты хранятся в `localStorage`, а вкладки одного браузера синхронизируются
через `BroadcastChannel`. Этого достаточно, чтобы проверить всю партию —
откройте четыре вкладки и сыграйте сами с собой.

## Подключение Firebase

1. Создайте проект в [console.firebase.google.com](https://console.firebase.google.com).
2. Включите **Authentication → Sign-in method → Anonymous**.
3. Создайте базу **Cloud Firestore** (production mode).
4. Скопируйте параметры веб-приложения в `firebase/config.js`:

```js
export const firebaseConfig = {
  apiKey: '...',
  authDomain: '...',
  projectId: '...',
  storageBucket: '...',
  messagingSenderId: '...',
  appId: '...',
};
```

5. Разверните правила и сайт:

```bash
npm i -g firebase-tools     # единственный инструмент, и тот глобальный
firebase login
firebase use --add          # выберите свой проект
firebase deploy --only firestore:rules,hosting
```

Приложение само определит, что конфигурация заполнена, и переключится с
локального режима на Firestore. Если Firebase окажется недоступен, игра
не упадёт: она вернётся в локальный режим и скажет об этом.

## Как устроена партия

Ведущего в игре нет — его роль исполняет **хозяин комнаты**. Только его клиент
переводит фазы, разбирает ночь и объявляет победителя; остальные читают
документ комнаты и отправляют свои действия. Один автор записей вместо гонки
десяти — отсюда и предсказуемость, и экономия обращений к базе.

```
лобби → знакомство с ролью → ночь → утро → голосование → ночь → … → итог
```

Ночь заканчивается досрочно, как только сходили все активные роли;
голосование — как только проголосовали все живые. Если хозяин закрыл вкладку,
права переходят к тому, кто вошёл в комнату раньше остальных, и партия
продолжается с той же точки.

### Схема данных

```
rooms/{code}                    настройки, состояние партии, hostId
rooms/{code}/players/{uid}      имя, аватар, готовность, жив ли
rooms/{code}/secret/{uid}       роль игрока — читает только он сам
rooms/{code}/secret/_engine     память ведущего: яд, прошлые выборы
rooms/{code}/actions/{round_uid} ночное действие — читает только ведущий
rooms/{code}/votes/{round_uid}  голос, создаётся один раз
rooms/{code}/chat/{id}          общий чат
rooms/{code}/chatMafia/{id}     закрытый чат мафии
```

Состояние партии целиком лежит в документе комнаты, поэтому у клиента всего
две постоянные подписки: комната и список игроков. Голоса и чат
подписываются только на своих экранах и отписываются при уходе.
Идентификаторы `round_uid` заодно решают две задачи: подменить автора
невозможно, а проголосовать дважды не даст запрет на обновление документа.

### Безопасность

Клиенту не доверяет ни одна проверка, которая что-то решает:

- роль лежит в закрытом документе — чужую не прочитать даже из консоли;
- ночные действия видит только хозяин комнаты;
- голос создаётся один раз, обновление запрещено правилами;
- чужие карточки игроков правит только ведущий;
- переписка мафии вынесена в отдельную коллекцию с отдельным правилом чтения;
- отравления и прошлые выборы хранятся в служебном документе `_engine`,
  недоступном игрокам.

## Оформление

Игра выглядит как ночной город после дождя: не чёрный, а глубокий
сине-графитовый фон, подсвеченный фиолетовой и лазурной вывеской. Светлой
темы нет намеренно — вторая тема («Дымка») тоже тёмная, просто теплее.

Всё оформление собрано из токенов в `styles/base.css`: цвета, стеклянные
слои, градиенты, свечения, радиусы, отступы, шкала кегля и кривые
анимаций. Меняете токен — меняется вся игра, точечных цветов в коде нет.

Фон — семь слоёв: небо, сияние, звёзды, три плана застройки и туман.
Слои двигаются с разной скоростью вслед за курсором или наклоном телефона
(`src/ui/atmosphere.js`) и меняют цвет вместе с фазой партии: лобби —
фиолетовый, ночь — синий, утро — золотой, голосование — красный.

Поверхности сделаны матовым стеклом: полупрозрачный фон, размытие,
светлая линия по верхней кромке и мягкая тень. Неон появляется только на
активных элементах — иначе он перестаёт что-либо значить.

Звук собран из синусоид в одной тональности, проходит через фильтр и
короткую реверберацию. Отдельные звуки: наведение, нажатие, подтверждение,
уведомление, ошибка. Проверить набор целиком можно в настройках кнопкой
«Проверить звук».

При включённом системном «уменьшении движения» параллакс не запускается,
а анимации выключаются.

Подпись автора в нижней панели лежит в `LINKS` в `src/core/config.js`.

## Как добавить роль

1. Создайте `src/roles/inspector.js` по образцу соседей:

```js
export default {
  id: 'inspector',
  icon: '🔍',
  team: 'town',          // town | mafia | solo
  order: 45,             // место в очереди разбора ночи
  isBase: false,
  actsAtNight: true,
  knowsAllies: false,
  name: { ru: 'Инспектор', en: 'Inspector' },
  short: { ru: 'Ищет следы', en: 'Follows traces' },
  description: { ru: '…', en: '…' },
  actionLabel: { ru: 'Кого проверяете?', en: 'Who do you inspect?' },

  canTarget: (ctx, actorId, targetId) => targetId !== actorId,

  resolve(ctx, action) {
    ctx.note(action.actorId, { type: 'check', text: { ru: '…', en: '…' } });
  },
};
```

2. Добавьте импорт в `src/roles/index.js` — и всё: роль появится в настройках
   комнаты, в справочнике правил и в раздаче.

Контекст `ctx` даёт справки (`roleOf`, `teamOf`, `isAlive`, `visitorsOf`,
`previousChoice`) и намерения (`kill`, `heal`, `block`, `defend`, `guard`,
`poison`, `note`, `publish`, `cancelKillsBy`). Все намерения собираются и
считаются один раз в конце ночи в фиксированном порядке: блокировки → защита →
лечение → убийства → яд → проверки. Поэтому новая роль не ломает старые.

## Возможности

- Комнаты с пятизначным кодом, передача прав хозяина, возврат в свою партию.
- Настройки комнаты: до 15 игроков, состав ролей, время фаз, чат, открытие роли.
- 13 ролей: мирный, мафия, дон, доктор, комиссар, маньяк, телохранитель,
  любовница, адвокат, журналист, свидетель, шпион, отравитель.
- Ночные действия, утренняя сводка, голосование с живым счётом, авто-победа.
- Живой фон ночного города с параллаксом, реагирующий на фазу партии.
- Нижняя панель: состояние связи, число игроков в комнате, версия, автор.
- Два языка (RU/EN), две тёмные темы, звук с синтезом при отсутствии файлов,
  поддержка `prefers-reduced-motion`, адаптив от телефона до десктопа.

## Версии

Текущая версия указана в `src/core/version.js`. Там же `SCHEMA_VERSION` —
поднимайте его, если меняете структуру документов: старые комнаты в новой
схеме открывать нельзя.
